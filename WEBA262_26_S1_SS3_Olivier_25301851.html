6. Improvements

The following improvements were implemented based on testing results and self-assessment. Each change is marked in the code with //Change X comments. The old code is commented out above the replacement where applicable.

Change # Description Effect Located In
1 Cookie storage for player name and best score Previously only localStorage was used. Cookies now persist player name and best score across browser sessions with a 365-day expiration. Falls back to localStorage if cookies are empty. launcher.js
2 Dropdown for difficulty, radio buttons for theme, Save/Load/Reset buttons Previously used custom button groups for difficulty and theme selection. Replaced with a <select> dropdown for difficulty and <input type="radio"> for color scheme to meet form input requirements. Added dedicated Save, Load, and Reset buttons with settings persistence. launcher.js, index.html
3 Prompt dialog for name if not entered Previously the name was silently set to 'Anonymous'. Now uses prompt() to ask the player for their name before falling back to Anonymous. Also saves settings to localStorage and opens game.html as a separate popup window. launcher.js
4 Two-page architecture (game opens in separate window) Previously the game ran on the same page as the launcher. Now the launcher (index.html) saves settings to localStorage and opens game.html as a popup via window.open(). The game reads its settings from localStorage and saves scores back. game.js, game.html
5 Special mode with 6 colors, random flash speed, 90-degree rotation Previously only 4 colors with fixed timing. Special mode adds orange + purple, randomized show/pause timing (150-750ms / 100-400ms), and the Simon wheel rotates 90 degrees left/right every 5 rounds while the color sequence stays unchanged. game.js
6 Game-over overlay with proper cleanup Previously the game-over div had no backdrop or fixed positioning. Fixed with a full-screen fixed overlay with dark backdrop. All pending timeouts (timer interval and sequence timeout) are now cleared when the game ends, preventing stale callbacks from interfering. game.js, styles.css

7. Reflection

[Write a comprehensive reflection (approximately 1 page) covering the following topics:]

7.1 Technical Aspects

Throughout this project, I gained significant experience with JavaScript programming, particularly in areas such as event handling, DOM manipulation, and Web Audio API integration. One of the most challenging technical aspects was restructuring the application from a single-page architecture to a two-page architecture with data passing between pages via localStorage. This required careful planning to ensure that settings, scores, and game state were synchronized correctly across windows.

I also learned how to work with browser storage mechanisms — cookies for persistent data, localStorage for shared settings and leaderboard data, and sessionStorage for temporary game state. Each storage type has its own use cases and limitations, and choosing the right one was an important consideration.

The implementation of the Special mode with dynamic clip-paths for 6-color segments and CSS transform-based rotation taught me a lot about advanced CSS techniques and how to create visually engaging effects.

7.2 User Experience

Through the survey and focus group, I learned how important user feedback is in the development process. Small details that seemed trivial during development (such as the game-over screen visibility, button placement, and visual feedback) turned out to have a significant impact on the user experience. The splash text feature (random messages under the title) was particularly well received, adding personality to the game.

I also realized the importance of clear instructions and intuitive controls. The keyboard support (keys 1-6) was appreciated by users who preferred keyboard input over mouse clicks, and the volume slider gave users control over the audio experience.

7.3 Challenges and Solutions

One of the main challenges was debugging the game-over flow — early versions had issues where pending timeouts would fire after the game ended, making the sequence appear to continue. This was solved by tracking all timeout IDs and clearing them in the gameOver function. Another challenge was the leaderboard deduplication, which required careful filtering logic to ensure each player's highest score was kept while removing lower duplicates.

The restructuring to meet assignment requirements (6 files, two-page architecture, specific form inputs) was the most time-consuming but ultimately rewarding part of the project. It forced me to think about separation of concerns and maintainable code architecture.

7.4 General Overview

This project gave me a complete experience of the software development lifecycle — from initial concept and implementation, through testing and user feedback, to final improvements and documentation. I now have a deeper appreciation for both the technical and user-facing aspects of web application development.
